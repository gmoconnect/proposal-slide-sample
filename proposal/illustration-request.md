# 挿絵作成依頼書（デザイナー向け）

## プロジェクト概要

- **プロジェクト名**: クラウドネイティブアーキテクチャによるWebサービス基盤刷新のご提案
- **用途**: 官公庁向けプロポーザル提案書のスライド資料（Slidev / 16:9）
- **対象読者**: 官公庁の評価者（技術の非専門家を含む管理職）
- **デザインツール**: NanoBanana Pro
- **納品形式**: PNG
- **依頼点数**: 必須 8点 + 任意 2点 = 計 10点

> スライド本体はSlidevコンポーネント（テーブル・カード・フロー等）で構成済み。本依頼は**コンポーネントでは表現できない構造的ダイアグラム**のみを対象とする。

## 共通デザイン指示

### デザインスタイル

McKinsey / BCG コンサルティングファームのミニマルスタイルに準拠する。

- **清潔で権威的な印象**: 情報密度は高いが、整然としたレイアウト
- **禁止事項**: 3D効果、グラデーション、影(ドロップシャドウ)、装飾的な図形、アニメーション
- **線**: 細いヘアライン（1-2px）。ボックスの角丸は4px
- **余白**: 要素間に十分な余白を確保。詰め込みすぎない
- **Tufte原則**: データを表すインク量を最大化し、装飾を最小化する

### ill-03: 多層防御オニオンモデル

| 項目 | 内容 |

| ファイル名 | `ill-03-onion-defense.png` |

| キーメッセージ | WAF・GuardDuty・CloudTrailの多層防御により、エッジからデータ層まで自動的に保護する |

#### レイアウト

同心円（オニオンモデル）構成。外側から内側に向かって防御層を配置。右側に監視・監査パネル。

**同心円（外側→内側）:**

1. **最外層** — エッジ層（橙系枠 `#F59E0B`）

2. **第2層** — 認証層（橙系枠 `#F59E0B`）

3. **第3層** — コンピュート層（青系枠 `#3B82F6`）
   - サービス: ECS on Fargate（read-only FS、SSHなし）

4. **中心** — データ層（緑系塗り `#10B981`）

- GuardDuty（紫 `#8B5CF6`）— ラベル: 「監視層: 機械学習ベース脅威検出」

#### 補足ラベル

- 図タイトル: 「多層防御アーキテクチャ（オニオンモデル）」
